## 1. 技术规格对比表

| 维度 | TCA9535RTWR | CH32V003F4U6 固件实现（本仓库） |
|---|---|---|
| GPIO 数量 | 16 路（P0、P1 各 8） | 16 路（可映射至任意 16 个 MCU 引脚，受封装/复用限制） |
| I2C 地址 | 0x20–0x27（A2/A1/A0） | 固件可设定任意 7-bit 地址（默认 0x20） |
| 寄存器映射 | 0x00/0x01 输入；0x02/0x03 输出；0x04/0x05 极性；0x06/0x07 配置 | 完全复刻同地址与意义；支持寄存器指针自增与读清 INT 行为 |
| 上电默认 | 全部为输入（Configuration=0xFF,0xFF） | 相同（固件初始化时配置） |
| 中断 | INT 开漏，输入变化置位，读输入寄存器清除 | 由任一 GPIO 模拟开漏输出；固件检测输入变化置低，读输入寄存器释放 |
| I2C 速度 | 标准/快速：100/400 kHz | 支持作为从设备在 100/400 kHz 下工作（依主控时序） |
| 供电 | 1.65–5.5 V | 2.7–5.5 V（MCU 供电域） |
| I/O 容忍 | P 口 5V 容忍 | MCU GPIO 不保证 5V 容忍（需遵守 VDD） |
| 驱动能力 | 典型 IOL up to 25 mA/Pin（总电流限制见手册） | 建议 8 mA 等级（VOL≈0.4 V）；20 mA 时压降明显（见数据手册） |
| 待机功耗 | 低至 µA 级 | 取决于 MCU 时钟/待机策略；可用低功耗模式降低消耗 |
| 封装/面积 | 24-pin WQFN (RTW) 4×4 mm | 取决于 MCU 封装与外设引出及必要电路 |

注：TCA9535 具 5V 容忍 I/O；MCU 方案不可直接 5V 容忍，需电平约束或加保护。

## 1.1 功耗对比（关键数据）

- TCA9535RTWR（器件级，I/O 设为输入、无负载）
  - 工作电流 ICC（fSCL=400 kHz）：
    - VCC=5.5 V：典型 22 μA（最大 40 μA）
    - VCC=3.6 V：典型 11 μA（最大 30 μA）
    - VCC=2.7 V：典型 8 μA（最大 19 μA）
    - VCC=1.95 V：典型 5 μA（最大 11 μA）
  - 待机电流 ICC（fSCL=0、I/O 输入、无负载）：
    - VCC=5.5 V：典型 1.5 μA（最大 3.9 μA；若输入为 GND，最大 8.7 μA）
    - VCC=3.6 V：典型 0.9 μA（最大 2.2–4 μA）
    - VCC=2.7 V：典型 0.6 μA（最大 1.8–3 μA）
    - VCC=1.95 V：典型 0.6 μA（最大 1.5–2.2 μA）

- 基于 CH32V003F4U6 的“软扩展器”系统（MCU 运行以响应 I2C）
  - Run 模式：
    - VDD=3.3 V：FHCLK=48 MHz 时约 4.0–6.4 mA（典型，取决于时钟源与外设开关）
    - VDD=5.0 V：FHCLK=48 MHz 时约 5.1–9.0 mA（典型）
  - Sleep 模式（仅停 CPU，外设与时钟保持）：
    - VDD=3.3 V：约 1.0–4.7 mA（典型，取决于频率与外设）
  - Standby 模式（最低功耗）：
    - LSI off：约 7.6–8.0 μA（典型）；LSI on：约 9.1–9.4 μA（典型）
    - 注意：Standby 下 I2C 不可用，无法“随时可访问”，与 TCA9535 的在线低功耗不等价

结论要点：

- 需“随时可被主控访问”的在线扩展器：TCA9535 的功耗低 2–3 个数量级（μA～几十 μA），MCU 方案通常为 mA 量级。
- MCU 仅在 Standby（不可响应 I2C）才能接近 μA 级；若要在线响应，功耗会升至 mA 级。

数据来源：

- TI TCA9535 Datasheet（SCPS201E）电气特性表中 ICC（Operating/Standby）
- CH32V003 Datasheet V1.7 表 3-6（Run）、表 3-7（Sleep）、表 3-8（Standby）

### 1.1.1 3.3V 典型功耗对比（按在线可用场景）

- TCA9535（VCC=3.3V）
  - 工作（fSCL=400kHz、I/O为输入、无负载）：ICC 典型 ≈ 11 μA（数据表在 3.6V 为 11 μA，3.3V 处于其间，量级相当）
  - 待机（fSCL=0）：ICC 典型 ≈ 0.9 μA

- CH32V003 软扩展器（VDD=3.3V）
  - Run：典型 ≈ 4.0–6.4 mA（FHCLK=48MHz；保持 I2C 在线响应）
  - Sleep：典型 ≈ 1.0–4.7 mA（停CPU但外设/时钟保持；可在线）
  - Standby：典型 ≈ 7.6–9.1 μA（不可在线响应 I2C）

结论（3.3V）：

- 在线可用（可随时响应 I2C）下，TCA9535 约 10 μA 级，而 CH32V003 软方案为 mA 级，高约 100–1000×。
- 若系统允许离线休眠（不响应 I2C），MCU 可降至 μA 级 Standby，但功能上不等价于 TCA9535。

## 2. 优缺点分析与应用建议

优点（MCU 固件实现）：

- 灵活：可映射任意引脚，扩展功能（滤波、去抖、复杂逻辑）可在固件中叠加。
- 可定制：I2C 地址/寄存器行为/中断策略可自定义；可叠加安全策略（看门狗、CRC）。
- 物料弹性：在缺芯/成本波动时期，可用通用 MCU 替代专用 IO 扩展器。

缺点：

- 供电与电气边界：无 5V 容忍 I/O；驱动能力和 ESD/抗扰度通常不及专用器件。
- 功耗与复杂度：固件维护、时序一致性、测试/认证成本更高；待机功耗可能更大。
- 可靠性：需要严格的固件质量保障与看门狗；应对时序边界条件（NACK、ARLO、BERR、总线卡死）。

应用建议：

- 低速控制/状态采集、对 I/O 电平电流要求不高的场景可用 MCU 方案；
- 对 5V 容忍、强灌/拉电流、极低待机功耗、工业 ESD/EMI 要求高的量产长期方案优选 TCA9535；
- 批量生产时，若 BOM/面积关键、供应稳定，优先专用芯片；若需高度定制/功能复合，可选 MCU 方案。

## 3. 固件结构与使用

目录：firmware/ch32v003_i2c_gpioexp/

- include/i2c_gpioexp.h：API 头文件
- src/i2c_gpioexp.c：实现（I2C 从机+寄存器机理+GPIO 应用）
- examples/main_example.c：示例工程入口（需接入 WCH 外设库与启动文件）

初始化：

1) 连接 I2C1 至 PC1(SDA)/PC2(SCL)（默认复用）；
2) 调用 gpioexp_init(0x20) 设 I2C 7bit 地址；
3) gpioexp_set_pinmap() 传入 16 路 GPIO 映射；
4) gpioexp_set_int_pin() 指定一个 GPIO 用作 INT（固件以“拉低=中断”模拟开漏）。

寄存器兼容：

- 读写 0x02/0x03 输出寄存器会更新对应引脚输出；
- 读 0x00/0x01 输入寄存器会清除 INT；
- 极性寄存器仅影响输入读取值，不改变实际引脚电平；
- 配置寄存器位=1 表示输入，=0 表示输出（与 TCA9535 一致）。

## 4. 性能与边界

- I2C：从机在 100/400 kHz 下工作，事件中断服务中实现寄存器自增、ACK/NACK 处理；
- 输入轮询：提供 gpioexp_poll_inputs()，建议在主循环或定时中断中调用，检测变化并拉低 INT；
- INT 清除：读取输入寄存器后释放 INT（与 TCA9535 行为一致）；
- 电气：请遵守 MCU GPIO 驱动/电压限制，必要时外接晶体管/缓冲器驱动外设。

---

## 5. 成本与面积（粗略）

- TCA9535RTWR：单芯片、外围仅需上拉，面积最小，价格取决于市场（一般 0.4–0.9 USD 量级，浮动）。
- CH32V003F4U6：MCU+编程接口+时钟/电源去耦，面积/成本通常高于单 IO 扩展器，但可复用 MCU 资源做更多事。

---

## 6. 可靠性与抗干扰

- TCA9535：专用器件，ESD/瞬态/时序经过厂家深度验证，工业级稳健。
- MCU：需关注 I2C 异常（BERR/ARLO/AF）、总线卡死、复位恢复（POR），建议开启看门狗、上电时序管理、I2C 超时恢复。

---

## 7. 编译与测试指引（建议）

- 基于 MounRiver Studio 或任意支持 CH32V00x 的构建链；
- 链接 WCH 外设库头文件与启动文件；
- 将 examples/main_example.c 设为入口，接 I2C 主机测试：
  - 向 0x20 写入 0x06/0x07（配置）后写 0x02/0x03（输出）观察引脚；
  - 读取 0x00/0x01 验证输入与 INT 清除；
- 用示波器验证 I2C 时序与 INT 脉冲。

---

## 8. 注意事项

- MCU GPIO 非 5V 容忍；
- INT 由固件用 GPIO 模拟开漏，外部需上拉至 VDD；
- 输入读取的去抖/滤波可按需在固件叠加；
- 如果 I2C 总线较长/噪声大，注意上拉值与布线、EMC 处理。
