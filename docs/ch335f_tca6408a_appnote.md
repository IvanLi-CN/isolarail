## CH335F + TCA6408A 应用笔记

### 1. 适用范围与目标

- 适用器件：WCH CH335F（USB 2.0 HUB 控制器）、TI TCA6408A（8 位 I2C GPIO 扩展器，TCA6408ARGTR 封装）、ESP32-S3（MCU）
- 供电条件：系统 3.3V 单电源（VDD33=3.3V；TCA6408A VCCI=3.3V、VCCP=3.3V）
- 采用架构：TCA6408A 采集 CH335F 端口电源使能状态 PWREN#；由 MCU 基于电源开关芯片检测结果，经 TCA6408A 注入 OVCUR# 过流事件
- 目标：提供可复用的模块化连接方法、参数选型、初始化流程与调试清单，降低集成风险。

### 2. 关键结论与要点

- 方向约束：
  - CH335F 的 PWREN#x 是“输出”（低有效）。禁止任何外部主动源与之强驱动相连。
  - CH335F 的 OVCUR#x 是“输入”（低有效，内部上拉微安级）。可由 TCA6408A 拉低（或由系统其他电源管理逻辑拉低）。
- TCA6408A P 口能力：IOL≤25 mA、IOH≤10 mA；3.3V 阈值 VIH≈2.31V、VIL≈0.99V（0.7/0.3×VCC）。足以直接驱动/读取 CH335F 的 3.3V 逻辑。
- 推荐实现：
  - P0..P3 设为输入：读取 CH335F 的 PWREN1#..4#。
  - P4..P7 设为“开漏等效输出”：注入 OVCUR1#..4#（低有效）。
  - 实际过流由 MCU 判定，TCA 注入用于联动/上报及一致性测试。

### 3. 模块角色与参考连接框图（文字说明）

- MCU：I2C 主机；采集电源管理芯片/采样回路的过流判定结果，决定是否通过 TCA 注入 OVCUR#。
- TCA6408A：I/O 扩展——读取 PWREN#（输入），拉低 OVCUR#（输出）。
- CH335F：端口电源控制与 USB 协议主导；OVCUR# 输入接收过流。

### 4. 硬件实现

#### 4.1 供电与去耦

- TCA6408A：VCCI=3.3V、VCCP=3.3V；VCCI/VCCP 各自就近放置 0.1 µF + 1 µF 去耦。
- CH335F：VDD33=3.3V；若对外提供 USB VBUS，需 5V 电源与专用开关（非 3.3V）。仅做逻辑/数据时可不供 5V，但需符合产品策略与安全规范。

#### 4.2 I2C 上拉电阻（3.3V）

- 计算：
  - Rp(min) ≈ (VCC − VOL(max))/IOL（由器件下拉能力与目标 VOL 决定）。
  - Rp(max) ≈ tr/(0.8473×Cb)（Fast-Mode 取 tr=300 ns；Standard-Mode 取 1 µs）。
- 经验推荐：
  - 400 kHz、板内短线、Cb≈100–200 pF：2.2–3.3 kΩ。
  - 100 kHz 或较长总线、Cb≈200–400 pF：3.3–4.7 kΩ。
- INT：开漏，已连接至 ESP32-S3 GPIO16；4.7–10 kΩ 上拉至 VCCI，低有效（低电平触发）。

#### 4.3 地址/复位/中断引脚

- TCA6408A ADDR：接 GND → 7-bit 地址 0x20（或接 VCCP → 0x21）。
- TCA6408A RESET：如无主动复位需求，10 kΩ 上拉至 VCCI（或直接接 VCCI）。
- TCA6408A INT：开漏输出，上拉至 VCCI；可接至 MCU 中断用于状态变化提醒（可选）。
- CH335F RESET#/CDP：内部上拉。推荐使用"外部上拉 + MCU 开漏"方案（见 6.3 节）。

#### 4.4 与 CH335F 的信号映射（方案 A）

- TCA P0..P3（输入） ← CH335F PWREN1#..PWREN4#。
- TCA P4..P7（“开漏等效输出”） → CH335F OVCUR1#..OVCUR4#。
- 可在上述连接中加入 100–330 Ω 串联小电阻，提供 ESD/瞬态保护与轻阻尼（可选）。

#### 4.4.1 当前实物连接映射

- 供电/去耦：VCCP=3.3V、VCCI=3.3V，均就近各 100 nF 去耦（C117/C118）。
- 地址：ADDR→GND（I2C 7-bit 地址 0x20）。
- 端口映射：
  - P0 ← PWREN1#（输入）
  - P1 → OVCUR1#（输出，低有效注入）
  - P2 ← PWREN2#（输入）
  - P3 → OVCUR2#（输出，低有效注入）
  - P4 ← PWREN3#（输入）
  - P5 → OVCUR3#（输出，低有效注入）
  - P6 ← PWREN4#（输入）
  - P7 → OVCUR4#（输出，低有效注入）
- INT#：已引出，作为开漏信号需上拉至 3.3V（4.7–10 kΩ），或由 MCU 端内部上拉实现。
- RESET#：当前未接使用；保持默认亦可。若需更高健壮性，建议下版以 ~10 kΩ 上拉至 VCCI。

#### 4.6 走线与 SI/PI 建议

- I2C/SCL/SDA：短且对地参考，必要时 MCU 端串 22–33 Ω 阻尼。ESP32-S3 使用 GPIO8(SDA)/GPIO9(SCL)。
- PWREN#/OVCUR#：短走线、地参考，避免与高速 USB 差分线平行长距离耦合。
- 端口电源走线：按电流密度选线宽与过孔数，VBUS 与地大电流回路紧耦合；电容就近。

### 5. 软件配置（TCA6408A）

#### 5.1 关键寄存器（Command Index）

- 0x00：输入寄存器（Input，读）。
- 0x01：输出寄存器（Output，读/写；默认全 1）。
- 0x02：极性翻转（Polarity，默认 0）。
- 0x03：方向配置（Configuration，1=输入；0=输出；默认全 1）。

#### 5.2 初始化（示例，ADDR=0x20）

1) 配置方向（0x03）：P0/P2/P4/P6=1（输入），P1/P3/P5/P7=0（输出） → 0b0101_0101（0x55）。
2) 输出默认（0x01）：将 P1/P3/P5/P7 置 1（释放 OVCUR#，不告警）。
3) 可选：使能 INT 读取/中断驱动的事件处理（Polling 亦可）。

#### 5.3 “开漏等效”注入 OVCUR

- 需要注入过流（拉低）：将对应 Pn 设为输出并写 0。
- 需要释放为高：先写 1，再将对应 Pn 改为输入（高阻释放，由 CH335F 内部上拉拉高）。
- 注：在全 3.3V 且无外源强驱动“高”的前提下，也可采用持续推挽 1/0 控制；但“高阻释放”兼容性更佳。

#### 5.4 读取 PWREN# 状态

- 读取 0x00 并掩码 0x55（P0/P2/P4/P6）即可。PWREN# 为低有效，软件层面建议统一转换为“端口已使能=1”的应用语义。

#### 5.5 端口事件推荐流程（真实过流为准）

1) MCU 基于电源管理芯片/采样回路确认端口 n 过流。
2) MCU 经 TCA 将对应 OVCUR#x 拉低（注入），保证 CH335F 进入一致的保护/上报路径。
3) 按策略执行端口复位/再上电；恢复前释放 OVCUR#x（高阻）。

### 6. INT 与 RESET 使用

#### 6.1 INT 中断使用要点

- 物理：INT 为开漏、低有效；本设计已连接 ESP32-S3 GPIO16，并通过 4.7–10 kΩ 上拉至 VCCI（3.3V）。
- 触发：MCU 配置为“低电平”中断。
- 清除：读取 TCA6408A 输入寄存器 0x00（完成 ACK 后）即可清除本次中断源。
- 典型流程：INT 触发 → MCU 中断服务例程读取 0x00 → 根据映射比对 P0/P2/P4/P6 的 PWREN# 状态差异 → 如需，软件注入对应 OVCUR#（见 5.3）。

#### 6.2 TCA6408A RESET（低有效）

- 作用：复位寄存器与 I2C 状态机（全部 P 口回到“输入”）。
- 时序：tW≥4 µs；推荐若无需要，不接 MCU，直接上拉至 VCCI（10 kΩ）或直连 VCCI。
- 何时需要：
  - 需要一键恢复 IO 配置/总线恢复的系统场景。
  - 现场维护/异常 I2C 恢复策略的一部分。
  - 基于 TPS82130SILR PG 信号的电源管理系统中，确保器件在电源稳定后可靠启动。

#### 6.3 CH335F RESET#/CDP（低有效）

- 特性：引脚内部上拉。上电期间若被外部“驱动为高”，可能启用 CDP 充电功能并影响低功耗。
**推荐方案：上拉电阻 + MCU 开漏只下拉**

- 连接方式：
  - RESET#/CDP ← 4.7–10 kΩ 上拉至 3.3V
  - MCU 引脚配置为开漏输出（Open-Drain），默认为输入/高阻状态
  - 无需额外的单向隔离或延时电路

- 工作原理：
  - 上电瞬间：上拉电阻立即将节点拉高（RC 时间常数 ~200 ns），无需等待 MCU 启动
  - 正常运行：MCU 保持高阻，节点由上拉维持高电平
  - 需要复位：MCU 开漏拉低，保持数毫秒后释放回高阻

- 上拉阻值选择：
  - 4.7 kΩ：更快上升沿，更强驱动能力，拉低时功耗 ~2.3 mW
  - 10 kΩ：平衡选择，拉低时功耗 ~1.1 mW
  - 推荐：4.7 kΩ（考虑走线电容与抗干扰）

- 功耗分析：
  - 高电平（正常状态）：几乎无电流消耗（仅引脚泄漏 nA 级）
  - 低电平（复位期间）：I = 3.3V/R，仅在主动复位时短暂消耗

- MCU 配置要点（ESP32-S3）：
  - 初始化：配置引脚为开漏模式，默认释放（高阻状态）
  - 正常运行：保持高阻，由外部上拉维持高电平
  - 执行复位：拉低保持数毫秒（≥4 µs），然后释放回高阻
  - ESP32-S3 支持 GPIO_MODE_OUTPUT_OD（开漏输出）模式

- 优势：
  - 上电即为高电平，不依赖 MCU 启动时序
  - 避免 CDP 功能误触发
  - 电路简单，成本低
  - 功耗极低

### 7. 版本化验证清单（Bring-up）

- I2C：0x20/0x21 应答；可读写 0x03/0x01；上升沿符合 400 kHz（或 100 kHz）设定。
- PWREN# 读取：插拔/枚举时，TCA 0x00 低 4 位变化与示波/逻分一致。
- 注入过流：软件拉低对应 OVCUR#x，Hub 对应端口掉电/上报。
- 真实过流：MCU 判定过流 → 通过 TCA 注入 OVCUR#x，确认 Hub 行为符合预期。
- RESET（如用）：拉低 TCA/CH335F 复位后，系统能恢复到既定初始状态。

### 8. 物料与参数一览（建议起点）

- I2C 上拉：2.2–3.3 kΩ（400 kHz，板内短线）；或 3.3–4.7 kΩ（100 kHz/较长）。ESP32-S3 内部弱上拉 ~45 kΩ，外部上拉必需。
- INT 上拉（如用）：4.7–10 kΩ。
- 保护/阻尼（可选）：PWREN#/OVCUR# 串 100–330 Ω。
- TCA6408A RESET 上拉：10 kΩ（或直连 VCCI）。
- CH335F RESET#/CDP 上拉：4.7–10 kΩ（推荐 4.7 kΩ）。
- TCA 去耦：VCCI/VCCP 各 0.1 µF + 1 µF，就近。
- **电源管理相关（新增）**：
  - TPS82130SILR PG 上拉：4.7 kΩ 至 3.3V
  - ESP32-S3 CHIP_PU RC 延时：10 kΩ + 1 µF
  - I2C 器件复位控制：GPIO38 开漏输出 + 10 kΩ 上拉
